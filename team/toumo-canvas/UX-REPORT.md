### 👎 需要改进的地方

1. **空白画布无引导** - 首次打开是空白 Frame，不知道从哪开始
2. **状态概念不直观** - "Display State" vs "Functional State" 对新手困惑
3. **交互添加入口隐藏** - 找不到在哪里给元素添加点击事件
4. **底部状态栏太小** - 状态切换按钮不够明显

## 🔍 详细问题分析

### 1. 新手引导缺失 (严重)

**问题**: 打开应用后，面对空白画布不知道该做什么

**建议**:
- 添加 Welcome Modal 或 Onboarding Tour
- 提供示例项目一键加载
- 空白状态显示操作提示（如 "按 R 创建矩形"）

### 2. 状态管理概念复杂 (严重)

**问题**: PRD 中的 "Functional State" 和 "Display State" 概念对设计师不友好

**现状**:
- 底部有 "State 1" 标签
- 右侧有 "State Mapping" 面板
- 但两者关系不清晰

**建议**:
- 简化术语：用 "页面/场景" 代替 "Functional State"
- 用 "动画帧" 代替 "Display State"
- 添加可视化状态图预览

### 3. 交互添加流程不直观 (中等)

**问题**: 想给按钮添加点击事件，但找不到入口

**现状**:
- 需要先创建多个状态
- 然后在 InteractionManager 的 State Graph 中连线
- 再设置 Trigger 条件

**建议**:
- 选中元素后，右键菜单添加 "Add Interaction"
- 或在属性面板底部添加 "Interactions" 区域
- 提供常用交互模板（点击切换、悬停效果等）

### 4. 工具栏图标不够清晰 (轻微)

**问题**: 工具栏使用 Unicode 符号（↖ ▢ ○ T ✋），不够专业

**建议**:
- 使用 SVG 图标
- 添加工具提示（已有，但延迟太长）
- 考虑添加工具名称标签
