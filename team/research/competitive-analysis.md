# 竞品分析报告

> 更新时间: 2025-01-20
> 研究员: Momo (用户研究员)

## 概述

本报告分析四款主流交互动画设计工具，为 Toumo 的产品定位和差异化策略提供参考。

---

## 1. Figma

### 产品定位
协作式 UI 设计工具，动效功能作为原型设计的补充。

### 核心动画能力

**Smart Animate**
- 自动识别匹配图层，动画化位置、缩放、透明度变化
- 支持缓动选项：Linear、Ease In/Out、自定义贝塞尔曲线
- 适用场景：加载动画、视差滚动、手势交互、开关切换

**2024 更新**
- Auto Layout 增强：支持多行多列响应式布局
- 多重 After Delay：单个元素可设置多个延迟触发
- 响应式原型预览模式
- AI 辅助功能（部分暂停优化中）

### 优势
- 实时协作，团队友好
- 38% 市场份额，行业标准
- 丰富的插件生态

### 劣势
- **动效能力有限**：复杂动画需要大量"填充帧"
- **无状态机概念**：交互逻辑难以管理
- 不支持阴影动画、形状变形

### Toumo 机会点
| Figma 痛点 | Toumo 解决方案 |
|-----------|---------------|
| 复杂动画需要多帧 | 状态机 + 关键帧自动补间 |
| 交互逻辑混乱 | 可视化状态图 |
| 无法精确控制时序 | 属性级曲线覆盖 |

---

## 2. Principle

### 产品定位
Mac 专属的 UI 动画原型工具，专注流畅的交互动效。

### 核心能力

**时间轴动画**
- 可视化时间轴，精确控制缓动、时序
- 支持自定义曲线和弹簧动画

**Drivers 驱动器**
- 图层属性可"驱动"其他图层
- 实现复杂的联动动画

**交互触发**
- Tap、Drag、Scroll、Long Press、Hover
- 支持独立的水平/垂直滚动

### 优势
- 三帧预览直观
- 动画质量高，导出 MP4/GIF
- 从 Figma/Sketch 导入

### 劣势
- **Mac Only**：无法跨平台
- **无矢量变形**：形状动画受限
- **大型原型卡顿**：50+ 屏幕性能下降
- **无代码导出**

### Toumo 机会点
| Principle 痛点 | Toumo 解决方案 |
|---------------|---------------|
| Mac 限制 | Web 跨平台 |
| 交互逻辑有限 | 完整状态机 |
| 无法分享预览 | 一键生成链接 |

---

## 3. Origami Studio

### 产品定位
Meta 出品的免费原型工具，面向复杂交互和微交互设计。

### 核心能力

**节点式编程**
- Patch 系统：可视化连接输入输出
- 精确控制动画和交互逻辑
- 类似编程的灵活性

**设备能力**
- 访问陀螺仪、加速计、摄像头
- 触觉反馈、声音集成
- iOS 实时预览

**代码导出**
- 可导出代码片段供开发参考

### 优势
- 免费
- 微交互能力强大
- Google、Airbnb、Microsoft 在用

### 劣势
- **学习曲线陡峭**：需要编程思维
- **节点混乱**：大型项目难以管理
- **Mac Only**
- **性能问题**：大文件卡顿

### Toumo 机会点
| Origami 痛点 | Toumo 解决方案 |
|-------------|---------------|
| 节点复杂难懂 | 状态机更直观 |
| 学习成本高 | 5 分钟上手 |
| 界面混乱 | 清晰的三区布局 |

---

## 4. ProtoPie

### 产品定位
跨平台高保真原型工具，支持复杂交互和硬件集成。

### 核心能力

**高级交互**
- 无代码创建复杂动画
- 变量、条件逻辑、循环
- 3D 旋转、图层重排序

**硬件集成**
- 摄像头、麦克风、陀螺仪
- 支持自定义硬件组件
- 汽车、IoT、VR/AR 场景

**2024 更新**
- Auto Layout（Beta）：从 Figma 导入
- 预定义变量：$deviceHeight、$deviceWidth
- 性能优化：切换更流畅
- 企业功能：用户测试、外部设备连接

### 优势
- 跨平台（桌面、移动、Web）
- 交互能力丰富
- Figma/Sketch/XD 插件

### 劣势
- **界面臃肿**：功能多但不够精简
- **状态管理弱**：复杂流程难以组织
- **订阅制**：有付费门槛

### Toumo 机会点
| ProtoPie 痛点 | Toumo 解决方案 |
|--------------|---------------|
| 界面复杂 | 极简设计 |
| 状态管理弱 | 状态机核心 |
| 学习成本 | 渐进披露 |

## 关键洞察

### 1. 市场空白
没有一款工具将**状态机**作为核心设计范式。现有工具要么太简单（Figma），要么太复杂（Origami）。

### 2. 用户痛点共性
- 复杂交互难以管理和维护
- 动画时序控制不够精细
- 跨平台需求未被满足

### 3. Toumo 差异化定位
```
简单 ←————————————————————→ 强大
Figma    Principle    ProtoPie    Origami
              ↑
           Toumo
    (状态机让强大变简单)
```

### 4. 核心竞争力
1. **状态机驱动** - 独特的设计范式
2. **Web 原生** - 跨平台无门槛
3. **三帧预览** - 借鉴 Principle 的直观
4. **渐进披露** - 简单入门，深度可探索
